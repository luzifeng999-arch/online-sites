第一节课 · 入门基础

从零到会用

安装 CF · 浏览器插件 · 联网搜索 · 收集主题图并自动分类
🎯 让设计师完成 CF 安装 + 浏览器插件配置，掌握「联网收集主题图 → 自动下载 → 按风格归类到文件夹」完整 AI 辅助工作流
P1
下载 & 安装 CodeFlicker
0 – 10 min
  • 访问官网下载安装包
    codeflicker.com → 选操作系统 → 安装完成后登录账号
  • 界面认知三件事
    文件树 / 编辑区 / AI 对话侧栏，Chat = 问答，Composer = 帮你写改代码
  • AI 模型选择说明
    推荐新手用 Auto / Claude Sonnet，无需纠结，先跑起来最重要
⬇ CF 安装👤 账号登录🖥 界面认知
P2
安装浏览器插件 · 让 IDE 联网
10 – 25 min
  • 安装 Browser Tool 插件
    搜索「Browser」→ 安装 browser-tools-mcp，让 IDE 直接控制浏览器
  • 演示：AI 批量搜集主题图
    「帮我搜索 2025 春日促销、电商大促、3D 插画三种方向的活动主题图，分别整理成文件夹」→ AI 自动打开浏览器、搜索、筛选、汇总
  • 演示：浏览 Dribbble / Behance 收集主题图
    「找近 30 天最热的活动头图 / KV / Banner 设计，截图并描述风格亮点」
  • 理解工作原理
    Browser MCP 让 AI 读取网页文字 + 截图，相当于帮你看网并做笔记
🔌 MCP 插件🌐 联网搜索📸 网页截图
P3
用 AI 收集主题图 + 按风格分文件夹整理
25 – 50 min
1
确定收集主题

示例：「帮我收集春日促销主题图，适合电商活动页参考」，先定义用途、关键词和参考方向

2
AI 自动联网搜索 + 下载素材

「去站酷 / Behance / Dribbble 搜相关主题图，下载可参考的头图，优先收集高清、信息完整的案例」

3
按风格自动分类到指定文件夹

把搜到的主题图按「渐变科技 / 3D 插画 / 电商大促 / 极简高级」等风格分类，自动放进本地对应文件夹，设计师打开就能看

4
拆解这条工作流为什么值钱

重点讲清楚：以前是人肉搜图、保存、重命名、拖文件夹；现在是一句话完成「搜索 → 筛选 → 下载 → 分类」，直接变成团队可复用流程

🏆 课堂产出：得到一套按风格整理好的主题图库文件夹，设计师可直接打开参考，后续活动需求也能重复复用这条流程
★
扩展：IDE 能模拟人操作浏览器的任何动作
50 – 60 min

「帮我在草稿箱里填写这篇文章，然后点预览」

📷
截图对比

「截图当前页面，分析配色和字体使用方式，给出建议」

📊

「逐页翻阅 Behance，把移动端 App 案例整理成表格」

★ 核心认知：浏览器工具让 IDE 从「会写代码的工具」变成「会上网的助手」，替代大量重复网页操作
✏️
课后小练习（自愿，不打卡） 用今天学的浏览器插件，找一个你觉得设计不错的活动落地页（比如品牌联名 / 节日专题），截图保存下来，记录你觉得好看的地方。

第二节课 · 进阶实战

搭一个独立运行的设计灵感雷达

第一节学会了 IDE 能做什么 → 第二节用 IDE 搭一个真正能独立运行的应用：前端界面 + Python 后端 + 内置 Claude Vision 分析，离开 CF 也能用
🎯 主线：CF 生成「前端 + 后端 + Claude API 调用」完整应用骨架 → Skill A 写入多源采集 + Vision 分析逻辑 → Skill B 生成趋势周报模式 → Agent 跑通前后端联调 → Design Mode 改界面风格

⚠️ 关键认知：Skill 是告诉 CF 「帮我生成什么代码」，真正的 Claude 分析能力住在生成出来的应用里，不在 CF 里
SK1
✦ 搭完整应用骨架：前端 + 后端 + Claude API
0 – 20 min
核心认知：CF 生成的不只是一个页面，而是一个完整的独立应用——前端负责展示，后端负责调用 Claude Vision 分析，两者联调跑通后，离开 CF 也能独立运行。
1
说清楚完整应用的结构（0–3 min）

输入：「帮我搭一个设计灵感雷达应用，包含：① 前端 index.html：搜索栏、来源筛选、灵感卡片（含 AI 分析摘要字段）、收藏功能，风格参考 Linear；② 后端 server.py：Flask 服务，负责调 Dribbble API 抓数据、把图片发给 Claude Vision 分析、返回 JSON 给前端；③ .env 文件：存放 CLAUDE_API_KEY」

关键点：前端只负责展示，真正的 Claude 分析能力在后端，应用独立运行不依赖 CF。

2
配置 Claude API（3–8 min）

在 .env 文件里填入内部 Claude API 接口信息，这是平台真正的「大脑」：

# .env — 平台的 Claude API 配置 WQ_API_KEY=课上请填入自己的 Key CLAUDE_MODEL=ep-yi4t2l-1775115584174507438 CLAUDE_API_BASE=http://wanqing.internal/api/gateway/v1/endpoints

⚠️ .env 不上传 Git，Key 不共享。Claude 的能力住在这个应用里，不是 CF 里。

3
启动后端 + 打开前端（8–18 min）

python server.py 启动后端服务 → 浏览器打开 index.html → 点击「抓取灵感」，后端真实调用 Claude Vision，卡片渲染出有 AI 分析摘要的数据。

# server.py 核心调用（CF 自动生成）——OpenAI SDK + 图像理解 client = OpenAI(base_url=CLAUDE_API_BASE, api_key=WQ_API_KEY) POST /api/fetch # 触发采集 + 分析 → 调 Dribbble API 拿图片 URL → client.chat.completions.create(model, [{image_url}, {text:"分析..."}]) → 返回 JSON 给前端渲染
4
抛出关键认知（18–20 min）

「这个应用现在可以关掉 CF，独立运行。Claude 的分析能力在 server.py 里，不在 CF 里——这才是一个真正可以交付、可以给团队用的工具。」

🏗 前端骨架就位⚙️ 后端服务运行中🧠 Claude 能力内置
SK2
⭐ 接入能力：Skill A 灵感采集 + Skill B 趋势周报
20 – 40 min
核心认知：同一个平台，装不同 Skill = 不同能力。Skill A 让平台学会「多源采集 + Claude Vision 分析每张图」；Skill B 切换成「趋势归纳 + 输出周报」模式。换 Skill 不用改平台。

📄 Skill A：灵感采集分析（.codeflicker/skills/radar-collect.md）

# 灵感雷达采集 Skill # 能力：多源抓取 + Claude Vision 逐图分析 ## 触发方式 当我说「帮我抓今天的设计灵感」时启动 ## 数据源（按顺序采集） - Dribbble API → 热门 UI / 插画 / 品牌作品 - Behance API → 完整项目集，含设计说明 - Awwwards（Browser MCP）→ 获奖网站截图 - 站酷（Browser MCP）→ 国内设计趋势 目标：共采集 60–100 条原始数据 ## 分析步骤 1. 对每张图调用 Claude Vision： 提取主色调 / 布局结构 / 风格关键词 / 可借鉴点 2. 汇总所有分析结果写入 data.js 3. 页面卡片自动渲染：图片 + AI 分析摘要 + 风格标签 ## 输出 灵感卡片页面更新 + 采集摘要：N 条 / 热门风格 Top3

📄 Skill B：趋势周报（.codeflicker/skills/radar-report.md）

# 设计趋势周报 Skill # 能力：聚合分析结果 → 归趋势 → 输出报告 ## 触发方式 当我说「帮我生成本周设计趋势报告」时启动 ## 执行步骤 1. 读取 data.js 中已有的 Claude Vision 分析结果 2. 发给 Claude 做趋势归纳： - 色彩趋势（高频色值 + 配色风格） - 布局趋势（网格类型 / 信息密度） - 风格趋势（关键词频率统计） 3. 筛选最具代表性的 5–8 个案例 4. 输出 weekly-report.html ## 输出 weekly-report.html：结构化周报，可截图发给团队

⚡ 为什么比自己上网搜强

1
自己搜：逛 10 个网站，截图收藏，2 小时，得到一堆没分析的图
2
灵感雷达：触发一次，5 分钟，60–100 条，每张图都有 AI 分析摘要
3
Skill B 周报：「本周 65% 作品用暗底亮色」「圆角集中 12–16px」，有观点有数据
4
换 Skill = 换能力：再装一个「配色提取 Skill」，平台多一种分析维度，骨架不动
🌐 10+ 数据源Dribbble / Behance / Awwwards / 站酷 / 花瓣 / Lapa.ninja 等，国内外覆盖
👁 Claude Vision 分析不只是收集图片，AI 真的看懂每张图：配色、布局、风格关键词
� 两个 Skill 两种模式同一个平台：按需找灵感 or 输出周报，切换 Skill 即可
📦 长期可用资产每次运行积累数据，周报越出越有对比价值，不是一次性工具
⭐ 核心认知：工具的价值不是帮你收集，是帮你理解。Browser MCP 是腿，Claude Vision 是眼，Skill 是插件——三件事组合，才是一个真正有用的工具。
P2
Agent：前后端联调 + 自动测试验收
40 – 50 min
  • 一句话让 Agent 跑通前后端联调
    「帮我把灵感雷达完整跑通：启动 server.py，确认 /api/fetch 接口可用，触发一次采集，检查 Claude Vision 分析结果能否正确渲染到前端卡片，移动端适配正常」→ Agent 自主完成整个链路
  • 发现问题自动修复
    常见问题：跨域（CORS）报错、Claude API 返回格式和前端解析不匹配、图片 URL 无法访问 → Agent 找到原因、修复代码、重新验证
  • Diffs View 审查每一步改动
    Agent 改完代码后，Diffs View 逐条查看改了哪里。接受 / 拒绝，控制权在你手里——AI 做，你审
🤖 前后端联调📋 Diffs 审查📱 移动端验证
P3
Design Mode：把灵感雷达改成你自己的风格
50 – 60 min

平台能跑通了，数据也进来了，但 AI 生成的界面是通用的。最后 10 分钟，不看代码，框选改成你自己的风格——用设计师的眼光收尾。

  • ⌘+Shift+D 进入 Design Mode
    在内置浏览器里直接框选灵感卡片、标题、背景区域 → AI 识别你指的是哪个元素，等你的指令
  • 示例：「卡片背景改成深色，标签用彩色区分，头部加一个渐变 Banner」
    实时看到变化。改完就是你自己风格的工具——和别人的 AI 生成品完全不同
  • 完整闭环总结
    SK1 搭平台骨架 → SK2 接入抓取能力 → Agent 跑通整链路 → Design Mode 界面个性化 → 你有了一个真实可用的设计灵感工具
🏆 课堂最终产出：一个真实可运行的「设计灵感聚合平台」——会自动搜索网络，有自己的界面风格，可以每天打开用
✏️
课后可以试试 给你的灵感雷达加新能力，选一个方向：
· 写一个「配色提取 Skill」，让雷达自动输出每个案例的色板
· 把数据源换成 Muzli / Awwwards，看看抓回来什么不同内容
· 用 Design Mode 把界面改成你负责项目的品牌色，截图发出来

💡 你搭的不是一次性 demo，是一个每周都能跑、可以持续积累设计数据的雷达系统。
