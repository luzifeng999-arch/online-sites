学完你能做什么

两节课覆盖从零开始到用 IDE 搭建并拥有一个真实可用的设计工具平台的完整路径，不需要写代码。
第一节
从零上手 · 搜图归类 · 主题图库搭建
  • 安装 CodeFlicker
    完成安装、登录、界面认知，了解 Chat / Composer 的区别
  • 浏览器插件联网
    让 IDE 控制浏览器，自动搜索 Dribbble / Behance 收集设计灵感
  • AI 收集主题图并自动分类
    搜索资料 → 下载主题图 → 按风格归档到指定文件夹，直接给设计师参考
第二节
搭平台 · Skill 接入能力 · 界面优化
  • 用 IDE 搭一个 Web 平台
    一句需求描述，AI 直接生成可运行的设计灵感聚合工具骨架
  • Skill 接入外部能力
    给平台装上「联网抓取 Dribbble」的能力，Skill = 可随时换掉的能力插件
  • Agent 测试 + Design Mode 美化
    Agent 跑通整个链路并修复问题，再用 Design Mode 把界面改成你自己的风格
第一节课 · 入门基础

从零到会用

安装 CF · 浏览器插件 · 联网搜索 · 收集主题图并自动分类
🎯 让设计师完成 CF 安装 + 浏览器插件配置，掌握「联网收集主题图 → 自动下载 → 按风格归类到文件夹」完整 AI 辅助工作流
P1
下载 & 安装 CodeFlicker
0 – 10 min
  • 访问官网下载安装包
    codeflicker.com → 选操作系统 → 安装完成后登录账号
  • 界面认知三件事
    文件树 / 编辑区 / AI 对话侧栏，Chat = 问答，Composer = 帮你写改代码
  • AI 模型选择说明
    推荐新手用 Auto / Claude Sonnet，无需纠结，先跑起来最重要
⬇ CF 安装👤 账号登录🖥 界面认知
P2
安装浏览器插件 · 让 IDE 联网
10 – 25 min
  • 安装 Browser Tool 插件
    搜索「Browser」→ 安装 browser-tools-mcp，让 IDE 直接控制浏览器
  • 演示：AI 帮你搜集设计资料
    「帮我搜索 2025 年最新的 UI 设计趋势，整理成要点」→ AI 自动打开浏览器、搜索、汇总
  • 演示：浏览 Dribbble / Behance 收集灵感
    「找近 30 天最热的移动端 App 界面设计，截图并描述设计亮点」
  • 理解工作原理
    Browser MCP 让 AI 读取网页文字 + 截图，相当于帮你看网并做笔记
🔌 MCP 插件🌐 联网搜索📸 网页截图
P3
用 AI 收集主题图 + 按风格分文件夹整理
25 – 50 min
1
确定收集主题

示例：「帮我收集春日促销主题图，适合电商活动页参考」，先定义用途、关键词和参考方向

2
AI 自动联网搜索 + 下载素材

「去站酷 / Behance / Dribbble 搜相关主题图，下载可参考的头图，优先收集高清、信息完整的案例」

3
按风格自动分类到指定文件夹

把搜到的主题图按「渐变科技 / 3D 插画 / 电商大促 / 极简高级」等风格分类，自动放进本地对应文件夹，设计师打开就能看

4
拆解这条工作流为什么值钱

重点讲清楚：以前是人肉搜图、保存、重命名、拖文件夹；现在是一句话完成「搜索 → 筛选 → 下载 → 分类」，直接变成团队可复用流程

🏆 课堂产出：得到一套按风格整理好的主题图库文件夹，设计师可直接打开参考，后续活动需求也能重复复用这条流程
★
扩展：IDE 能模拟人操作浏览器的任何动作
50 – 60 min

浏览器里你能做的，AI Agent 都能做——而且更快更批量

▶
点击 / 导航

「打开 xxx 网站，点击第一篇文章，把正文内容复制下来」

📋
填写表单

「帮我在草稿箱里填写这篇文章，然后点预览」

📷
截图对比

「截图当前页面，分析配色和字体使用方式，给出建议」

📊
批量抓取数据

「逐页翻阅 Behance，把移动端 App 案例整理成表格」

★ 核心认知：浏览器工具让 IDE 从「会写代码的工具」变成「会上网的助手」，替代大量重复网页操作
✏️
课后小练习（自愿，不打卡） 用今天学的浏览器插件，找一个你觉得设计不错的活动落地页（比如品牌联名 / 节日专题），截图保存下来，记录你觉得好看的地方。

💡 第二节我们会把「搭一个设计工具平台 → 给它接入搜图能力 → 再自己优化界面」这件事完整跑一遍。
第二节课 · 进阶实战

搭一个自己的平台

第一节学会了 IDE 能做什么 → 第二节直接用 IDE 搭建一个设计工具平台，接入不同的能力，再优化它的界面
🎯 主线：从 0 搭一个「设计灵感聚合工具」的 Web 平台 → 用 Skill 给它接入联网搜索能力 → 用 Agent 跑通整个链路 → 用 Design Mode 把界面改成自己喜欢的风格
SK1
✦ 搭平台骨架：一句话生成可运行的 Web 工具
0 – 20 min
核心认知：IDE 不只能改代码——它能从 0 帮你生成一个完整的 Web 应用。一句需求描述，直接跑起来的平台就出来了。
1
说清楚你想要什么工具（0–3 min）

输入一句描述：「帮我做一个设计灵感聚合工具，有搜索栏、展示卡片区域、收藏功能，风格参考 Linear，用单个 HTML 文件实现」

关键点：说清楚「功能是什么 + 视觉风格参考 + 技术形式」，AI 就能一次生成可运行的骨架。

2
现场看平台跑起来（3–15 min）

AI 生成 HTML + CSS + JS → 在浏览器直接打开 → 是一个真实可操作的界面：有搜索栏、卡片展示区、收藏按钮，但数据还是空的。

<!-- 设计灵感聚合工具 · 骨架版 --> <!-- 功能：搜索栏 / 灵感卡片 / 收藏 / 过滤标签 --> <div class="app"> <header>设计灵感聚合 · 搜索栏</header> <div class="card-grid"> <!-- 灵感卡片区（空壳）--> <div class="card">等待接入数据...</div> </div> </div>
3
感知差距，引出下一步（15–20 min）

骨架好看，但是空的——没有真实数据。抛出问题：「怎么让这个工具自动帮我去搜 Dribbble 上的内容，填进卡片里？」 → 引出 Skill 接入能力。

🏗 平台骨架已就位📭 数据待接入
SK2
⭐ 接入能力：用 Skill 给平台装上「眼睛」
20 – 40 min
核心认知：Skill 是给平台接入外部能力的方式——让工具学会「自动搜索网络 + 提取内容 + 填充到界面」。不同的 Skill = 不同的能力插件，随时可以换。

📄 Skill 文件（.codeflicker/skills/inspiration-fetch.md）

# 灵感抓取 Skill # 能力：浏览器 MCP + 内容提取 + 写入平台数据 ## 触发方式 当我说「帮我抓取今天的设计灵感」时启动 ## 执行步骤 1. 打开浏览器搜索 Dribbble / Behance 最新热门 2. 提取 Top 10 作品：标题 / 作者 / 标签 / 截图描述 / 链接 3. 按 JSON 格式整理数据 4. 把数据写入平台的 data.js 文件，触发页面刷新 ## 输出格式 更新 data.js，页面卡片区自动渲染新数据 并在侧栏生成「本次抓取摘要」：共抓取 N 条 / 热门标签 Top3

⚡ 接入后的效果

1
SK1 骨架已就位：平台界面可运行，卡片区待填充
2
说「帮我抓取今天的设计灵感」 → Skill 自动打开浏览器 → 搜索抓取 → 写入 data.js
3
刷新平台页面：卡片区真实数据进来了——标题、标签、描述都填好了
4
换一个 Skill = 换一种数据源：把 Dribbble 换成 Pinterest / Awwwards，平台立刻有新能力
� Skill = 能力插件每个 Skill 是一个独立能力，随时接入 / 换掉，平台不用重写
🌐 浏览器 MCP 自动抓取不需要 API key，直接模拟人在网上操作，搜索 + 截图 + 提取
🔄 多能力叠加再加一个「配色提取 Skill」「字体分析 Skill」，平台能力翻倍
� 平台 = 长期工具不是一次性文件，是你自己的工具，下次打开继续用、继续升级
⭐ 核心认知：这就是「用 IDE 搭平台」的本质——骨架是平台，Skill 是插件，每加一个 Skill 就多一种能力
P2
Agent：让平台整个跑通，自动测试验收
40 – 50 min
  • 一句话让 Agent 接管整个链路
    「帮我把这个平台跑起来，触发一次灵感抓取，检查卡片有没有正常渲染，移动端能不能用」→ Agent 自主完成：启动 → 抓取 → 检查 → 修复
  • 发现问题自动修复
    比如卡片图片加载失败、移动端卡片溢出 → Agent 找到原因 → 自己写修复代码 → 重新验证
  • Diffs View 审查每一步改动
    Agent 改完后，在 Diffs View 逐条查看：改了哪里、为什么改。接受 / 拒绝，控制权在你手里
🤖 链路自动测试📋 Diffs 审查📱 移动端验证
P3
Design Mode：把平台界面改成自己的风格
50 – 60 min

平台能跑通了，但 AI 生成的界面不是你的审美？最后 10 分钟，不看代码，直接上手框选改。

  • ⌘+Shift+D 进入 Design Mode
    在内置浏览器里直接框选灵感卡片、标题、背景区域 → AI 识别你指的是哪个元素，等你的指令
  • 示例：「卡片背景改成深色，标签用彩色区分，头部加一个渐变 Banner」
    实时看到变化。改完就是你自己风格的工具——和别人的 AI 生成品完全不同
  • 完整闭环总结
    SK1 搭平台骨架 → SK2 接入抓取能力 → Agent 跑通整链路 → Design Mode 界面个性化 → 你有了一个真实可用的设计灵感工具
🏆 课堂最终产出：一个真实可运行的「设计灵感聚合平台」——会自动搜索网络，有自己的界面风格，可以每天打开用
✏️
课后可以试试 把今天搭的平台继续升级：再加一个 Skill（比如「配色提取」或「竞品截图对比」），让你的工具多一种能力。或者用 Design Mode 把界面改成你的品牌色，截图分享出来。

💡 你搭的不是一次性文件，是一个可以持续迭代的属于你自己的设计工具。
