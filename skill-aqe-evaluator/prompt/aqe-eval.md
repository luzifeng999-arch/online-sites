# DIR-05 · 评测类 Skill 专属 Checklist

> 此提示词用于评测「对其他 Skill 产出进行打分/评测」类 Skill
> 在 system.md 的基础上追加使用

## 特殊准入要求（评测类必须回答）
- [ ] 评测对象类型（UI截图 / 图片文件 / 代码 / 文档）？评测输入必须精确定义
- [ ] 评分输出格式：数值分（0-10）/ 等级（ABCDE）/ 布尔（pass/fail）/ 维度报告？
- [ ] 人工标注基准：有 ≥ 20 条人工标注样本作为 LLM-as-Judge 的校准基准？

## AQE-D 品牌一致性（评测类核心维度，权重45%）
重点检查：
- **评分稳定性**：同一输入多次调用，评分结果一致（不大于 ±1.0 分）
- **评分维度可解释**：每个评分维度有明确的「1分是什么样 / 5分是什么样 / 10分是什么样」的描述

## AQE-E 可访问性（评测类特殊规则，权重30%）
重点检查：
- **Ground Truth 准确率 ≥ 85%**：用 10 条已知答案样本测试，评测 Skill 的打分与人工判断一致率 ≥ 85%
- **反馈可执行**：评测输出的问题描述包含「具体问题 + 建议操作」，不是纯分数

## AQE-B 色彩美学（评测报告的呈现方式，权重25%）
重点检查：
- **评测报告含维度分布**：总分 + 各维度得分 + 失分维度排名 + 典型失败案例
- **多人标注一致性 κ ≥ 0.7**（如涉及多人标注）

## 评测类 Skill 特殊说明

评测类 Skill 本身无视觉输出（输出是数字或文字报告），因此：
- AQE-A 视觉层级：**不适用**（权重 0%）
- AQE-C 版式规范：**不适用**（权重 0%）
- AQE-B 色彩美学（权重25%）：主要评测「评测报告」本身的视觉呈现，若报告是纯文字则降权至 15%
- AQE-E（权重30%）：主要评测「评测准确率」和「反馈质量」，而非传统可访问性

## 评测类失败模式

### KF-EVAL-001：评分不稳定
- **症状**：同一输入，10次调用得分从3.2到7.8，标准差 >2.0
- **修复**：在 Skill prompt 中增加「温度设置 temperature=0，评分前先输出思考链」

### KF-EVAL-002：反馈不可执行
- **症状**：报告写「视觉效果较差，需要改进」，没有具体行动建议
- **修复**：要求 Skill 必须输出「具体问题 + 建议操作 + 预期改善值」三元素

### KF-EVAL-003：标注基准缺失
- **症状**：没有 Ground Truth 样本，无法验证评测准确率
- **修复**：准入前必须准备 ≥ 10 条「输入+标准答案」对，综合准确率 ≥ 85% 才能上线
