## AQE-A 视觉层级专属检查

| 检查项 | 失分标准 | BLOCK? |
|--------|---------|--------|
| 字号层级 ≥ 3 级，对比比例合理（建议 20/14/12px 或等比），一眼可区分主次 | 字号层级 < 2级，或字号差异 < 2px | — |
| 视觉动线清晰：最重要信息在视觉焦点位置（通常左上或中央），次要信息不与主要信息竞争注意力 | 视觉焦点位置空白或放置次要信息 | — |
| 在最低分辨率下布局不崩溃：B端 ≥ 1280px，H5 ≥ 375px，无截断/叠压/越界 | 在1280px下有元素截断或叠压 | — |

## AQE-B 色彩美学专属检查

| 检查项 | 失分标准 | BLOCK? |
|--------|---------|--------|
| 主色系 ≤ 5 个色相，强调色 ≤ 2 个，整体色彩和谐无冲突 | 超过5种色相，或强调色超过3个 | — |
| 色彩情感与使用场景匹配（商务场景用蓝/灰，创意场景可鲜艳） | 配色与场景情感明显不匹配 | — |

## AQE-C 版式规范专属检查

| 检查项 | 失分标准 | BLOCK? |
|--------|---------|--------|
| 间距体系统一：所有间距使用 4px 或 8px 的倍数，同类元素间距相同，无随机值（7px/13px等） | 存在非4/8倍数的间距值 | — |
| 对齐无偏差：所有元素有明确对齐基准（左对齐/居中/右对齐），无浮动偏移 | 元素存在无规律偏移 | — |
| 留白充足：内容区域不超过页面 70%，边距合理，无「满屏堆砌」感 | 内容密度 > 70% | — |

## AQE-E 可访问性专属检查（强制）

| 检查项 | 失分标准 | BLOCK? |
|--------|---------|--------|
| 🔴 文字对比度 ≥ 4.5:1（WCAG AA） | 对比度 < 4.5:1 → E1 < 5.0 | **BLOCK** |
| 🔴 状态不依赖颜色独立表达：错误/成功/警告状态有文字标注或图标，灰度截图仍可理解 | 状态仅靠颜色区分，无文字/图标 → E2 < 5.0 | **BLOCK** |
| 可交互元素有 focus 样式（Tab 键遍历可见），按钮有 hover/active 状态 | 无 focus 样式，无 hover 状态 | 打回 |

## 典型已知失败模式（UI方向）

### KF-UI-001：随机间距值
- **触发条件**：CSS 未使用变量系统，手写每个 margin/padding 值
- **症状**：margin: 7px; padding: 13px; gap: 17px 混杂
- **根因**：没有明确的间距约束指令
- **修复指令**：在 Skill description 中加入「所有间距使用 4px/8px 整数倍，推荐值：4/8/12/16/24/32/48px」
- **防重现规则**：准入时检查 CSS 是否定义了 --space-1 到 --space-8 等 token

### KF-UI-002：对比度不足
- **触发条件**：使用浅灰色文字放在白色背景
- **症状**：`color: #999` 或 `color: #aaa` 作为正文颜色
- **根因**：AI 倾向于使用「柔和」色，未意识到无障碍标准
- **修复指令**：在 Skill prompt 中规定「辅助文字颜色 ≥ #767676（白底），不使用 #999 以下的灰」
- **防重现规则**：准入时 AQE-E1 必须使用 WebAIM Contrast Checker 验证

### KF-UI-003：移动端布局崩溃
- **触发条件**：Skill 未指定响应式要求
- **症状**：H5 场景下（375px）出现横向滚动条、文字截断
- **根因**：默认按桌面端（1280px）生成
- **修复指令**：在 Skill description 中加「必须适配 375px 最小宽度，使用 max-width + padding 而非固定宽度」
