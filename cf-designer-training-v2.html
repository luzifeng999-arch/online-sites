第一节课 入门基础
01

从零到会用

安装 CF · 浏览器插件 · 联网搜索 · 写文章
🎯 课程目标：让设计师完成 CF 安装 + 浏览器插件配置，掌握「网页浏览收集设计灵感 → 写设计文字内容」的完整 AI 辅助工作流
P1
下载 & 安装 CodeFlicker（CF）
0 – 10 min
  • 访问官网下载 CF 安装包
    codeflicker.com → 选操作系统版本 → 安装完成后登录账号
  • 初次启动与界面导览
    认识三个核心区域：文件树 / 编辑区 / AI 对话侧栏，了解 Chat 与 Composer 的区别
  • AI 模型选择说明
    推荐新手使用 Auto / Claude Sonnet，无需纠结模型选择，先跑起来是重要的
⬇ CF 安装 👤 账号登录 🖥 界面认知
P2
安装浏览器插件 · 让 IDE 联网
10 – 25 min
  • 安装 Browser Tool 插件
    搜索「Browser」→ 安装 browser-tools-mcp，这个插件让 IDE 能直接控制浏览器
  • 实操演示：让 AI 帮你搜集设计资料
    在 Composer 中输入「帮我搜索 2025 年最新的 UI 设计趋势，整理成要点」→ AI 自动打开浏览器、搜索、读取页面内容汇总
  • 实操演示：浏览 Dribbble / Behance 收集灵感
    「打开 dribbble.com，找近30天最热的移动端 App 界面设计，截图并描述设计亮点」
  • 理解工作原理：AI 怎么「看」网页
    简单解释：Browser MCP 让 AI 读取网页的文字内容 + 截图，相当于帮你看网并做笔记
🔌 MCP 插件 🌐 联网搜索 📸 网页截图
P3
用 AI 搜集资料 + 写一篇设计文章
25 – 50 min
  • Step 1：确定写作主题
    示例主题：「2025 年 UI 设计 10 大趋势」，在 Composer 输入主题，让 AI 提出文字结构建议
  • Step 2：AI 自动搜索 + 整理素材
    指令：「基于这个大纲，帮我搜索每个趋势的相关资料和案例，整理好来源和要点」
  • Step 3：生成文章初稿
    「根据以上素材，帮我写一篇适合发布在设计社区的 2000 字文章，语气专业但易读，加入具体案例」
  • Step 4：多轮对话精修文章
    「第3条案例换成 Apple Vision Pro 的 UI 设计」「结尾加一个对设计师的行动建议」—— 体验对话式迭代
🏆 课堂产出：每位学员完成一篇完整设计文章草稿，可直接发布到即刻、设计师网站等平台
★
扩展：IDE 能模拟人操作浏览器的任何动作
50 – 60 min

这一节告诉设计师：浏览器里你能做的，AI Agent 都能做——而且更快更批量

点击 / 导航 「打开 xxx 网站，点击第一篇文章，把正文内容复制下来」
📋
填写表单 / 输入内容 「帮我在草稿箱里填写这篇文章，然后点预览」
📷
截图 / 视觉对比 「截图当前页面，分析配色和字体使用方式，给出改进建议」
📊
批量抓取数据 「逐页翻阅 Behance 搜索结果，把所有移动端 App 案例的标题和作者整理成表格」
★ 核心认知：浏览器工具让 IDE 从「会写代码的工具」变成「会上网的助手」，可以替代大量重复的网页操作工作
✏️
课后作业
独立完成：用 CF + 浏览器插件，搜集一个你感兴趣的设计话题的资料，写一篇 1000 字左右的设计笔记并发布到任意平台（即刻 / 掘金 / 公众号 / 朋友圈均可）。

第二节课 进阶实战
02

从会用到出品

Design Mode · Skill 实战 · Agent 指挥 · 独立发布
🎯 课程目标：掌握 Design Mode 可视化编辑、写属于自己的第一个 Skill、用 Agent 多任务编排能力，最终独立产出一个可部署的真实作品（网页 / 工具 / 原型）
P1
Design Mode：用眼睛指挥 AI 改界面
0 – 15 min
  • 什么是 Design Mode
    ⌘+Shift+D 进入设计模式，在内置浏览器中用鼠标框选任意 UI 元素 → AI 自动加入对话上下文
  • 实操：框选 UI 元素 + 对话修改
    框选一个按钮 → 对话「把这个按钮改成圆角，颜色改成珊瑚红，加一个微妙的投影」→ 实时看到变化
  • Shift 框选多个元素批量修改
    框选一整个导航栏 →「把整个导航改成毛玻璃风格，背景半透明，border-bottom 改成渐变实线」
🎨 Design Mode 🎯 精准框选 ⚡ 批量修改
P2
从设计稿到真实网页：完整转化流程
15 – 30 min
  • 截图上传 Figma 设计稿 → 生成 HTML
    在 Composer 中上传设计稿截图 →「根据这张设计稿，生成对应的 HTML+Tailwind CSS 页面，保持视觉一致」
  • Design Mode 对齐视觉细节
    生成后用 Design Mode 框选不对的地方逐一修正，不需要看代码，全程自然语言操作
  • 响应式适配 + 交互动效一键加上
    「帮这个页面加上移动端响应式布局」「hover 时卡片轻微上浮，加半透明投影过渡动画」
🔑 关键认知：设计师不再需要等「开发」，自己就能把设计稿变成真实可运行的 HTML 页面
SK
✦ 写你的第一个 Skill
30 – 45 min
  • Skill 是什么？30 秒说清楚
    把一段常用指令固化成「工具」——下次说触发词，AI 自动按流程跑，写一次，复用 N 次
  • 演示一个现成 Skill：周报生成
    说「帮我写周报」→ Skill 自动触发 → 问答填写工作内容 → 金字塔结构输出，结论先行
# 周报生成 Skill

## 触发方式
当我说「帮我写周报」时启动

## 执行步骤
1. 询问：本周完成了哪些事（列条目即可）
2. 询问：遇到了哪些问题
3. 按金字塔原则重新组织，结论先行

## 输出格式
核心结论（2句）/ 完成事项 / 问题 / 下周计划

每人选一个场景，当场改写并验证 →

场景Skill 方向触发词
活动设计 活动 H5 需求文档生成 帮我写活动需求
数据分析 数据分析报告生成 + 可视化 帮我分析这份数据
素材生成 文字模板参数配置 Skill 帮我生成文字模板
文档规范 需求文档规范化（金字塔原则） 帮我规范这份文档
周报 / 汇报 散乱笔记 → 结构化汇报大纲 帮我写周报
✦ 产出：每人完成属于自己工作场景的 Skill，说触发词就能复用
P3
Agent 编排：指挥 AI 完成复杂任务
45 – 55 min
  • 什么是 Agent 模式
    Agent 模式下 AI 会自主规划步骤、创建文件、运行代码、查看结果并自我纠正，设计师只需描述目标
  • 实操：用 Agent 做一个完整的 Landing Page
    「帮我做一个设计师作品集 Landing Page：顶部 Hero 区 / 作品展示 Grid / 关于我 / 联系方式，风格简洁的深色系」
  • 多 Agent 并行：同时处理多个任务
    开多个 Composer 窗口，一个生成页面，一个写文案，一个搜索竞品参考，三路并行互不干扰
  • 审查 Diffs View：看 AI 改了什么
    每次 Agent 完成任务后，在 Diffs View 里查看变更内容，选择接受 / 拒绝，保持对结果的控制权
🤖 Agent 模式 📋 Diffs 审查 ⚡ 并行任务
P4
独立出品：发布你的第一个真实产品
55 – 60 min
  • 快速部署到公网（Vercel / GitHub Pages）
    「帮我把这个页面部署到 Vercel，生成一个可以分享的链接」—— 设计师第一次拥有真实的线上产品
  • 进阶：做自己的设计工具
    示例：「帮我做一个色板管理工具，可以输入颜色名称，自动生成对应的色板，支持导出 Tailwind 配置」
  • A/B 变体快速出包
    「把这个 Landing Page 生成 3 个配色变体：科技蓝 / 温暖橙 / 奢华深色，每个都能独立访问」
🏆 课堂产出：每位学员完成一个有真实访问链接的网页作品，可以发朋友圈展示
✏️
课后作业
独立完成：用 Agent 模式制作一个「设计师专属小工具」（色板、字体预览、组件库展示网站均可），部署上线分享链接。
